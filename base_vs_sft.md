## 6. [ml] Explain gradient descent as if I am a beginner.

- Source: `curated`
- Check: beginner-friendly explanation, not dry memorization

### Base

Certainly! Gradient descent is an optimization algorithm used to find the minimum of a function. Imagine you're trying to find the lowest point on a hilly landscape. The function we're trying to minimize is like that landscape, and the goal is to find the valley where the function's value is the smallest.

In the context of machine learning, this function often represents the error between our model's predictions and the actual data. The "gradient" in gradient descent refers to the slope of the function at any given point. It tells us which direction we need to move to reach the minimum more quickly.

Here’s how it works step-by-step:

1. **Initialization**: Start at a random point (or a point close to the minimum). This is your initial guess for the minimum.

2. **Compute the Gradient**: Calculate the slope (gradient) of the function at your current position. This tells you the direction in which the function increases most rapidly.

3. **Update Rule**: Move in the opposite direction of the gradient. The size of this step is determined by a parameter called the learning rate.

### SFT

Gradient Descent is an optimization algorithm used to minimize a function. It works by iteratively adjusting the parameters of the model in the direction of the negative gradient of the cost function. The learning rate determines how much weight to adjust at each step. The process continues until the cost function reaches a minimum. Gradient Descent can be implemented using batch gradient descent, stochastic gradient descent, or mini-batch gradient descent. Batch gradient descent uses all the training data for each iteration, while stochastic gradient descent uses only one sample. Mini-batch gradient descent uses a small random subset of the training data. Gradient Descent is widely used in machine learning algorithms such as linear regression, logistic regression, and neural networks. It is important to choose the right learning rate and number of iterations to ensure convergence to the optimal solution. Gradient Descent is also sensitive to local minima and may converge to a suboptimal solution. To overcome this, techniques like momentum, adaptive learning rates, and regularization can be used. Gradient Descent is an efficient and powerful optimization algorithm that has been widely used in many machine learning applications.

- **Precision** is calculated as: Precision = True Positives / (True Positives + False Positives)
- **Recall** is calculated as: Recall = True Positives / (True Positives + False Negatives)

Recall matters more when it's crucial to find all relevant items, even if some non-relevant items are included. For example, in medical diagnosis systems, high recall is important because missing a serious condition could be life-threatening. In contrast, in spam detection systems, where false positives (non-spam emails) can cause inconvenience but not harm, precision might be more critical.

For example, in an array:
- If I want to access the 10th element, I can calculate its position directly.
- In a linked list, I would need to start from the head and follow the pointer 9 times to reach the 10th element.

Linked lists are more flexible and can grow or shrink dynamically, but they have higher time complexity for random access. Arrays provide faster access to elements but require fixed size and cannot be resized easily.

- **GET /users/123**: Retrieves details of user 123.
- **POST /users**: Creates a new user record with provided data.

The key aspects of REST include:
- **Statelessness**: Each request from the client to the server must contain all the information necessary to understand and process the request.
- **Hypertext Orientation**: The API's response includes links to related resources, allowing clients to navigate through the system.
